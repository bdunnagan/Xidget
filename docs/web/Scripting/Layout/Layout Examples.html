Layout Examples

Overview

When writing layout scripts it is essential to understand which xidgets have a preferred size. It is only necessary to specify one side of a xidget if the corresponding dimension is defined. The size parameter specifies one or both dimensions of a xidget. A dimension is left unspecified by setting it to -1. A xidget may be invisible if its height is left unspecified and either its top or its bottom side does not have an attachment.

Another important consideration is to decide whether the size of the containing form should be set to accomodate the sizes of the xidgets it contains, or whether the xidgets should be sized to fit within the form. The former is called "packing" the form.

In general, if a layout gets too complex, it can be simplified by dividing up the contents into smaller forms and using straight-forward layout scripts for those forms.

Vertical Layouts

In the following layout, the xidgets are positioned from top to bottom using the default spacing. The preferred width of each xidget is used and the xidgets are aligned on the left side of the form. The form height is set to accomodate the content of the form.

<form> <layout> <layoutAlignLeft/> <layoutTopBottom/> <layoutPackY/> </layout> </form>

In the following layout, the xidgets are positioned from top to bottom as before, but the left and right sides of the xidgets are attached to the form - the xidgets fill the form horizontally. As before, the form height is set to accomodate the content of the form.

<form> <layout> <layoutFillX/> <layoutTopBottom/> <layoutPackY/> </layout> </form>

The next layout positions the xidgets on a vertical grid.

<form> <layout> <layoutFillX/> <layoutGridY/> </layout> </form>

Horizontal Layouts

The following example

<form> <layout> <layoutTopBottom/> <layoutFillX/> <layoutPackY/> </layout> <!-- other xidgets here --> <form> <layout> <layoutRightLeft/> <layoutAlignTop/> <layoutPackY/> </layout> <button> <label>'Ok'</label> </button> <button> <label>'Cancel'</label> </button> </form> </form>
