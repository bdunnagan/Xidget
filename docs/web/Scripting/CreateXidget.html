createXidget

Overview

The createXidget script element creates and binds a xidget. The location of the xidget configuration and the context to which the xidget will be bound are provided to the script. The location of the xidget configuration is identified by an xpath which can be defined in three ways: in the text of the element, in the xidget attribute, or in the xidget child element. The context is defined in the context attribute or child element. If the context is not defined then the current script context is used.

Any xidget can be created with the createXidget script element. If the xidget is an window xidget then a new window is created. In this case, the assign attribute should contain the name of the variable that will store the instance so that it can be destroyed later. If the xidget configuration is not an window xidget then the parent attribute, or child element, determines the xidget container to which the newly create xidget will be parented.

The event processing loop for the widget toolkit is entered after the main script completes. Thus, the main script can create more than one window xidget. The main script must complete, however, for the window(s) to become visible.
