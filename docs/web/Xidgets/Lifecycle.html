The Xidget Lifecycle

Xidget Creation

Xidget creation consists of the following stages: Creation, Initialization, Construction, Binding and Visibility. A xidget is created by calling the createXidget script element. A xidget is associated with a single xpath context that is supplied to the createXidget script, and the xpaths in the xidget configuration are bound to this context when the xidget is bound.

The Creation stage involves instantiating an implementation of the IXidget interface based on a configuration element. The mapping of the implementation of IXidget to a configuration element may be defined in the xidget base package, in the xidget toolkit implementation, or in a user-defined plugin.

Once the xidget class has been instantiated, the onOpen script is called. At this point, the xpaths within the xidget configuration have not yet been bound to the xidget context. This means, among other things, that the script can create xml fragments and assign them to variables that are referenced by xpaths in the xidget configuration.

After the onOpen script is called, the widget hierarchy is built, but the widget hierarchy is not made visible until after the xidget is bound to its context. Thus, a widget will be populated based on its xidget xpaths before it becomes visible.

Xidget Destruction

Similarly, the xidget destruction consists of the following stages: Visibility, Binding, Destruction, Finalization and Deletion. These stages can be understood by symmetry with the process of creation.
