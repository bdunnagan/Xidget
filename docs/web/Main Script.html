Main Script

A xidget application is started by invoking the JVM with the platform runnable jar and passing the file-system path of an xml file containing a xidget script. This script is called the main script as it is analagous to the main method of a Java application.

The main script is like any other script. It consists of script elements that are responsible for creating the application data-model and top-level application xidget.

Running A Xidget Application

The following example shows the main script for a "Hello, World!" application.

<assign name="message">'Hello, World!'</assign>
<print>$message</print>

This script would be executed by running the following command from the command line. The jar file, MyJar.xml, should be an executable jar containing all of the class files in the JAHM, Xidget and XidgetSwing projects. The argument to the org.xidget.swing.Main class is a path relative to the root of the jar file. So, the following example assumes that the main.xml script is at the root of the jar file.

java -jar MyJar.xml main.xml

Creating The Application Xidget

The top-level xidget for an application is created by calling the createXidget script element.
