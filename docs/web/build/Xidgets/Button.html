Button

Overview

The button xidget creates one of several types of buttons: push-button, check-button, toggle-button or radio-button depending on the value of the type parameter.

Push Button

The following example shows a push-button:

<button> <label>'Push Me'</label> <onPress> <print>'Button Pushed!'</print> </onPress> </button>

Check Button

In the following example, a check-button is created and its state is stored in the $value variable. The state will be one of the two strings: "true" or "false".

<button type="check"> <label>'Check Me'</label> <source>$value</source> </button>

Toggle Button

In the following example, a toggle-button is created and its state is stored in the $value variable. The state will be one of the two strings: "true" or "false".

<button type="toggle"> <label>'Toggle Me'</label> <source>$value</source> </button>

Radio Button

Radio buttons are grouped together by proximity. All buttons of type "radio" that appear consequetively in the configuration will belong to the same group and will be mutually exclusive. To create consequetive groups of radio buttons, place each group in a form. Giving each form a title is a simple way to identify the group.

Currently, the radio-button model is somewhat too manual. A simpler model will be supported in the future. The following example demonstrates two radio-buttons that operate correctly even when the selected option is set programmatically:

<button type="radio"> <label>'Option 1'</label> <source>$value = 'Option 1'</source> <onPress> <set source="'Option 1'">$value</set> </onPress> </button> <button type="radio"> <label>'Option 2'</label> <source>$value = 'Option 2'</source> <onPress> <set source="'Option 2'">$value</set> </onPress> </button>
